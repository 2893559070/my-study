  • 线程安全

    • 同步代码块 (锁对象)

      • 格式 让每个线程有自己的锁 synchronized(任意对象 | this) { 多条语句操作共享数据的代码 }
        • 使用this 可能会出现重复问题
        • 任意对象必须是 唯一的
        • 共享数据记住 需要开启 static 属性
        • synchronized
              // 同步方法
    private synchronized void synchronizedMethod() {
        // 执行代码
    }

    // 同步代码块
    synchronized (this) {
        // 执行代码
    }

    // 同步静态方法
    private static synchronized void synchronizedMethod() {
        // 执行代码
    }

    // 同步静态代码块
    synchronized (类名.class) {
        // 执行代码
    }
      • 默认情况下是打开的，只要有一个线程进去执行代码了，锁就会关闭
      • 当线程执行完出来了，锁才会自动打开
      • 同步的好处与弊端
        • 好处：解决了多线程的数据安全问题
        • 弊端：线程很多的时候，每个线程都会去判断同步上的锁，很耗费资源，无形中会降低程序的运行效率

    • Lock 锁 Lock是接口，不能直接实例化，这里曹勇它的实现类 ReentrantLock来实例化

      • void lock() 获得锁
      • void unlock 释放锁
            private ReentrantLock Lock = new ReentrantLock();
    
    try{
        // 获得锁
        Lock.lock()

        // 执行代码
    }catch(*** e) {
        // 捕获异常
    }finally {
        // 释放锁
        Lock.unlock()
    }
      • 死锁 两个或者多个线程互相有对象所需要的资源，导致这些线程处于等待状态，无法前往执行
            Object obj1 = new Object();
    Object obj2 = new Object();

    // 线程一

    // 线程二
